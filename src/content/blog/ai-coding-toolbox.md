## 写在前面

2026 年了，AI 编程工具已经不是"有没有用"的问题，而是"怎么搭配用"的问题。

我试过很多工具，最终留下来这五个，它们各司其职，覆盖了从编码、调试到日常沟通的完整工作流。而驱动这些工具的，是智谱的 GLM-5.1 和 GLM-4.7 模型。

这篇文章不是测评，是一个普通开发者的真实使用记录。

## 1. Claude Code — 终端里的编程搭档

**是什么：** Anthropic 推出的 CLI 编程代理，直接在终端里运行，能读写代码、执行命令、操作 Git。

**我的用法：**
- 重构代码时让它处理文件级的改动
- 写完代码让它跑测试、修 bug
- 复杂的 Git 操作（rebase、cherry-pick）直接交给它

**为什么喜欢：**
- 真正理解代码上下文，不是简单的补全
- 能跨文件操作，一次改十几个文件不会漏
- 终端原生，不用切窗口

**不足：**
- 需要 Anthropic API，费用按 token 计算
- 复杂推理建议用 Opus 模型，但 Opus 真的贵

> Claude Code 像一个坐在你旁边的高工，你跟他说需求，他直接动手改代码。

## 2. Antigravity — Google 的 AI IDE

**是什么：** Google 出品的 AI 集成开发环境，把 AI 能力深度嵌入到编辑器的每个角落。

**我的用法：**
- 快速原型开发——从想法到可运行代码，几分钟搞定
- 代码审查和优化建议
- 学习新框架时让它生成示例代码

**为什么喜欢：**
- IDE 深度集成，AI 体验比插件式（如 Copilot）流畅得多
- Google 的多模态能力——可以直接贴截图让它分析
- 免费额度对个人开发者够用

**不足：**
- 还在快速迭代，偶尔有 bug
- 生态不如 VS Code 成熟

> Antigravity 像一个全栈搭档，你负责想，它负责写。

## 3. CMUX — AI Agent 的终端管家

**是什么：** manaflow-ai 出品的 Ghostty 终端增强工具（GitHub 13.7k ⭐），专为 AI 编程 Agent 设计。垂直标签页 + Agent 状态通知，让多 Agent 协作变得清晰可控。

**我的用法：**
- 同时跑 Claude Code、Open Claw、Hermes，每个 Agent 一个标签页
- Agent 完成任务时收到桌面通知，不用盯着终端看
- 垂直标签在宽屏上比水平标签好用太多

**为什么喜欢：**
- 专为 AI Agent 工作流设计，不是事后加的功能
- Ghostty 底层快如闪电，渲染延迟几乎为零
- 和 tmux 完全不同的交互范式——更现代、更直觉

**不足：**
- 目前只支持 macOS（基于 Ghostty）
- 需要适应新的快捷键体系

> CMUX 解决了一个核心问题：同时管理多个 AI Agent 时的混乱感。

## 4. Open Claw — 开源 AI 编程平台

**是什么：** GitHub 224k ⭐ 的超人气开源 AI 编程助手。可以接入各种大模型，完全本地运行，数据不出你的电脑。

**我的用法：**
- 用智谱 GLM-5.1 作为主力模型
- 日常编码辅助——补全、重构、写测试
- 需要隐私保护的项目，用 Open Claw 跑本地模型

**为什么喜欢：**
- 完全开源，可以自己定制
- 模型自由——想用什么模型就用什么，不被锁定
- 社区活跃，插件生态丰富

**不足：**
- 开源项目的通病：文档有时跟不上代码更新
- 配置灵活也意味着上手成本高一些

> Open Claw 让你拥有自己的 AI 编程助手，不需要把代码发给任何人。

## 5. Hermes — 全能 AI 生活助手

**是什么：** NousResearch 出品的本地 AI Agent（GitHub 59k ⭐），不只是编程，而是全能型的。能操作电脑、连接微信、管理文件、搜索网页……是一个真正"住在你电脑里"的助手。

**我的用法：**
- 微信里直接和它语音对话，用中文聊需求
- 让它帮我搭建博客（就是你现在看到的这个 Hermes 日记）
- 自动化日常操作——文件整理、博客发布、GitHub 管理
- 定时任务——cron job 让它定期帮我检查东西

**为什么喜欢：**
- 继承本地权限，零配置就能用 GitHub、文件系统
- 微信原生语音消息回复，体验像和真人聊天
- 跨会话记忆——它记得我叫什么、偏好什么

**不足：**
- 需要自己部署，对非技术用户有门槛
- 依赖大模型 API，对话会产生费用

> Hermes 不只是编程工具，它是真正融入日常生活的 AI 伙伴。

## 关于模型：为什么选智谱 GLM

说到模型，我的主力是**智谱的 GLM-5.1 和 GLM-4.7**。

**GLM-5.1**（主力）：
- 中文理解和生成能力强，代码质量高
- 推理能力出色，复杂逻辑不容易出错
- API 价格友好，日常使用没有经济压力

**GLM-4.7**（备用 / 轻量任务）：
- 速度快，适合简单的补全和问答
- 作为 GLM-5.1 的经济替代，处理不需要深度思考的任务

我的策略很简单：
- **需要深度推理**（重构、架构设计）→ GLM-5.1
- **日常补全和快速问答** → GLM-4.7
- **需要顶级推理**（偶尔）→ Claude Opus

> 不必迷信最贵的模型。智谱的 GLM 系列在中文编程场景下性价比极高。

## 工具搭配的艺术

五个工具不是孤立的，它们是这样配合的：

```
日常开发
├── CMUX（终端环境，统一管理）
│   ├── Claude Code（复杂重构 + Git 操作）
│   ├── Open Claw + GLM-5.1（日常编码辅助）
│   └── Antigravity（快速原型 + 代码审查）
│
生活/自动化
└── Hermes（微信对话 + 博客管理 + 定时任务）
```

核心原则：**让每个工具做它最擅长的事。**

## 写在最后

工具在快速进化，今天的选择可能半年后就变了。但有一点不变——**理解工具的边界，比追求"最强工具"更重要**。

Claude Code 擅长深度编码，但不擅长日常对话；Hermes 擅长全能辅助，但不是专业 IDE；Antigravity 体验流畅，但生态还在成长。

找到适合自己的组合，把精力放在创造上，而不是工具本身。

> 工具是手段，不是目的。最好的工具，是你用得最顺手的那个。
